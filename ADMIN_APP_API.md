## Firestore Collections

### `businesses/{slug}`

The primary document for each discovered business. Slug is generated by
`generateSlug(name)` in `src/lib/reportStorage.ts` (lowercase, hyphens, stripped specials).

```typescript
// Top-level fields
{
  name: string;                          // "Bosphorus Turkish Cuisine"
  address: string | null;                // "123 Franklin Ave, Nutley, NJ 07110"
  officialUrl: string;                   // "https://bosphorusnutley.com"
  zipCode: string;                       // "07110" — ALWAYS a top-level field, never derived
  coordinates: { lat: number; lng: number } | null;
  createdAt: Timestamp;                  // server timestamp, set once
  updatedAt: Timestamp;                  // updated on every write

  // --- Enriched fields (top-level, written by writeDiscovery) ---
  // These are queryable directly on the document for admin filtering/sorting.
  // They are ALSO mirrored inside `identity` below for backwards compatibility.
  phone: string | null;
  email: string | null;
  hours: string | null;
  googleMapsUrl: string | null;
  socialLinks: {
    instagram?: string;
    facebook?: string;
    twitter?: string;
    yelp?: string;
    tiktok?: string;
    grubhub?: string;
    doordash?: string;
    ubereats?: string;
    seamless?: string;
    toasttab?: string;
  };
  logoUrl: string | null;                // GCS URL, never base64
  favicon: string | null;
  primaryColor: string | null;           // "#0f172a"
  secondaryColor: string | null;
  persona: string | null;               // "Old School Jersey Diner"
  menuUrl: string | null;               // URL to menu page
  competitors: { name: string; url: string; reason?: string }[];

  // --- Backwards-compat mirror (same fields nested under identity) ---
  identity: {
    phone: string | null;
    email: string | null;
    hours: string | null;
    googleMapsUrl: string | null;
    socialLinks: { /* same shape as top-level socialLinks */ };
    logoUrl: string | null;
    favicon: string | null;
    primaryColor: string | null;
    secondaryColor: string | null;
    persona: string | null;
    menuUrl: string | null;
    competitors: { name: string; url: string; reason?: string }[];
  };

  // Per-agent latest outputs (written by writeAgentResult)
  // Keys: "margin_surgeon" | "seo_auditor" | "traffic_forecaster" | "competitive_analyzer"
  latestOutputs: {
    [agentName: string]: {
      score: number | null;              // 0-100
      summary: string | null;
      reportUrl: string | null;          // GCS public URL to HTML report
      agentVersion: string;              // e.g. "1.0.0"
      runAt: Timestamp;
      // Agent-specific promoted KPIs (flat, no nesting):
      // margin_surgeon:        total_leakage, menu_item_count
      // seo_auditor:           (scores stored as score field)
      // traffic_forecaster:    peak_slot_score
      // competitive_analyzer:  competitor_count, avg_threat_level
    };
  };

  // CRM state (written by writeInteraction)
  crm?: {
    status: 'outreached' | 'responded' | 'archived';
    outreachCount: number;
    lastOutreachAt: Timestamp;
    lastReportShared: string;            // report URL
    respondedAt?: Timestamp;
    archivedAt?: Timestamp;
    archiveReason?: 'no_response' | 'not_interested' | 'converted';
  };
}
```

**Rules:**
- No arrays that grow over time (no `reports[]`, no `analyses[]`)
- No binary blobs — images go to GCS, store URL only
- `zipCode` is always top-level, never parsed from address at query time
- Use `update()` with dotted paths for nested fields; `set({merge:true})` only for top-level/new docs

## BigQuery Tables (`hephae-co-dev.hephae.*`)

All tables are append-only. Historical data lives here; Firestore stores only current state.
Writes use DML INSERT via `bqInsert()` in `src/lib/db/bigquery.ts` (not streaming).

CRM event log. Written by `writeInteraction()`.

| Column             | Type      | Description                                    |
|--------------------|-----------|------------------------------------------------|
| `interaction_id`   | STRING    | `{event_type}-{slug}-{timestamp}`              |

| `zip_code`         | STRING    | 5-digit zip, nullable                           |

## API Endpoints

All endpoints accept `POST` with `Content-Type: application/json`.

### Discovery & Chat

#### `POST /api/chat`
Conversational chat with Gemini. Supports business location via `locate_business` tool call.

```typescript
// Request
{
  messages: { role: 'user' | 'model'; text: string }[];
  context?: {                            // Optional — injected as system instruction
    businessName?: string;
    address?: string;
    seoReport?: { overallScore, sections, summary };
    marginReport?: { overall_score, menu_items, strategic_advice };
    trafficForecast?: { summary, daily };
    competitiveReport?: { market_summary, competitors, recommendations };
  };
}

// Response (normal)
{ role: "model"; text: string }

// Response (business located — triggers discovery in frontend)
{
  role: "model";
  text: string;
  triggerCapabilityHandoff: true;
  locatedBusiness: BaseIdentity;         // { name, address, coordinates, officialUrl }
}
```

#### `POST /api/discover`
Runs full discovery pipeline: parallel swarm (Menu, Social, Maps, Theme, Competitors).

```typescript
// Request
{ identity: BaseIdentity }

// Response: EnrichedProfile
// (BaseIdentity + primaryColor, secondaryColor, logoUrl, persona, menuUrl,
//  socialLinks, phone, email, hours, googleMapsUrl, competitors, etc.)
// Also includes reportUrl (GCS link to profile HTML report)
```

#### `POST /api/track`
Lead capture: logs search queries and captures emails.

```typescript
// Route 1: Log initial query
{ query: string }  →  { success: true, id: string }

// Route 2: Attach email to existing search
{ id: string, email: string }  →  { success: true }
```

### Capability Endpoints

#### `POST /api/analyze` — Margin Surgery
```typescript
// Request
{
  url: string;
  enrichedProfile: EnrichedProfile;      // without menuScreenshotBase64
  advancedMode: boolean;                 // false = fast estimation, true = live benchmarks
}
// Fast path: triggers when enrichedProfile.officialUrl is present.
//   Screenshots menuUrl if available, otherwise falls back to officialUrl.
// Slow path: no enrichedProfile.officialUrl → runs legacy Locator + Profiler.

// Response: SurgicalReport
{
  identity: BusinessIdentity;
  menu_items: MenuAnalysisItem[];        // each has: item_name, current_price, recommended_price,
                                         //   price_leakage, competitor_benchmark, commodity_factor,
```

## Admin App — Planned Features

### 1. Zip Code Deep Research Trigger
Trigger discovery + all capabilities for every business in a given zip code.
- Input: `zipCode` (5-digit string)
- Flow: Query Google Places for restaurants in zip → run `/api/v1/discover` for each → run all 4 capability endpoints → results land in Firestore + BQ automatically
- Status tracking: poll `businesses` collection filtered by `zipCode`

### 2. Business Browser
List all discovered businesses from Firestore `businesses/{slug}`.
- Filterable by: `zipCode`, `crm.status`, agent completion (has `latestOutputs.{agent}`)
- Sortable by: `updatedAt`, `latestOutputs.margin_surgeon.score`, etc.
- Detail view: show all `latestOutputs`, link to each HTML report via `reportUrl`

### 3. Zip Code Dashboard
Aggregate view across a zip code.
- BQ query: `SELECT * FROM analyses WHERE zip_code = @zip ORDER BY run_at DESC`
- Show: avg margin score, avg SEO score, total businesses discovered, leakage heatmap
- Cross-zip comparison table

### 4. CRM Pipeline View
Track outreach status across all businesses.
- Firestore query: `businesses` filtered by `crm.status`
- Columns: business name, zip, status, outreach count, last outreach date, responded?
- Actions: trigger follow-up email, archive business

### 5. Agent Run History
Full audit trail from BigQuery.
- Table: `analyses` — filter by agent_name, business_slug, date range
- Table: `discoveries` — show discovery timeline
- Table: `interactions` — CRM event log

## Useful Queries for Admin

### Firestore: All businesses in a zip code
```typescript
const snap = await db.collection('businesses')
  .where('zipCode', '==', '07110')
  .orderBy('updatedAt', 'desc')
  .get();
```

### Firestore: Businesses missing a specific agent run
```typescript
// No native "field does not exist" query — fetch all in zip, filter client-side
const snap = await db.collection('businesses')
  .where('zipCode', '==', '07110')
  .get();
const missing = snap.docs.filter(d => !d.data().latestOutputs?.seo_auditor);
```

### BigQuery: Leakage leaderboard for a zip code
```sql
SELECT business_name, business_slug, score, total_leakage, menu_item_count, run_at
FROM `hephae-co-dev.hephae.analyses`
WHERE zip_code = '07110' AND agent_name = 'margin_surgeon'
ORDER BY total_leakage DESC
LIMIT 20
```

### BigQuery: All agent runs for a business
```sql
SELECT agent_name, agent_version, score, summary, run_at
FROM `hephae-co-dev.hephae.analyses`
WHERE business_slug = 'bosphorus-turkish-cuisine'
ORDER BY run_at DESC
```

### BigQuery: Discovery history for a zip code
```sql
SELECT business_name, business_slug, zip_code, run_at
FROM `hephae-co-dev.hephae.discoveries`
WHERE zip_code = '07110'
ORDER BY run_at DESC
```

### BigQuery: CRM funnel metrics
```sql
SELECT
  event_type,
  COUNT(*) as count,
  COUNT(DISTINCT business_slug) as unique_businesses
FROM `hephae-co-dev.hephae.interactions`
WHERE zip_code = '07110'
GROUP BY event_type
ORDER BY count DESC
```

## Integration Testing Infrastructure

### Cloud Run Job: `integration-tests`

A Cloud Run Job that runs the full integration test suite (97 tests across 5 levels)
against live Gemini APIs. Uses the same Docker image as `hephae-forge`.

| Component          | Value                                             |
|--------------------|---------------------------------------------------|
| Job Name           | `integration-tests`                               |
| Image              | Same as `hephae-forge:latest`                     |
| Service Account    | `hephae-forge@hephae-co-dev.iam.gserviceaccount.com` |
| Memory / CPU       | 2Gi / 2 cores                                     |
| Task Timeout       | 900s (15 min)                                     |
| Secrets            | `GEMINI_API_KEY`                                  |
| Report Destination | `gs://everything-hephae/test-reports/{run-id}/`   |

### Test Levels

| Level | File                          | What it tests                        | Needs Browser |
|-------|-------------------------------|--------------------------------------|---------------|
| 1     | `test_google_search.py`       | Grounding-verified search results    | No            |
| 2     | `test_locator_agent.py`       | Business identity resolution         | No            |
| 3     | `test_discovery_pipeline.py`  | Full pipeline (7 state keys)         | Yes           |
| 4     | `test_firestore_structure.py` | Firestore doc structure validation   | Yes           |
| 5     | `test_hallucination_check.py` | LLM output accuracy vs ground truth  | Yes           |

### Scripts

```bash
# One-time: create the Cloud Run Job
./scripts/deploy-test-job.sh

# Trigger tests on Cloud Run (waits, prints report URL)
./scripts/trigger-cloud-tests.sh

# Check last execution status + report URL
./scripts/trigger-cloud-tests.sh --status

# Run locally with HTML report
./scripts/test-integration.sh --report

# Run locally, specific levels
./scripts/test-integration.sh --level 1 --level 2
```
