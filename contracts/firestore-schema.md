## `businesses/{slug}`

The primary document for each discovered business. Slug is generated by
`generateSlug(name)` in `web/backend/lib/report_storage.py` (lowercase, hyphens, stripped specials).

```typescript
{
  name: string;                          // "Bosphorus Turkish Cuisine"
  address: string | null;                // "123 Franklin Ave, Nutley, NJ 07110"
  officialUrl: string;                   // "https://bosphorusnutley.com"
  zipCode: string;                       // "07110" — ALWAYS a top-level field, never derived
  coordinates: { lat: number; lng: number } | null;
  createdAt: Timestamp;                  // server timestamp, set once
  updatedAt: Timestamp;                  // updated on every write

  // --- Enriched fields (top-level, written by writeDiscovery) ---
  phone: string | null;
  email: string | null;
  hours: string | null;
  googleMapsUrl: string | null;
  socialLinks: {
    instagram?: string;
    facebook?: string;
    twitter?: string;
    yelp?: string;
    tiktok?: string;
    grubhub?: string;
    doordash?: string;
    ubereats?: string;
    seamless?: string;
    toasttab?: string;
  };
  logoUrl: string | null;                // GCS URL, never base64
  favicon: string | null;
  primaryColor: string | null;           // "#0f172a"
  secondaryColor: string | null;
  persona: string | null;               // "Old School Jersey Diner"
  menuUrl: string | null;               // URL to menu page on their website
  menuScreenshotUrl: string | null;     // GCS URL to JPEG screenshot of menu page
  menuHtmlUrl: string | null;           // GCS URL to raw HTML of menu page
  competitors: { name: string; url: string; reason?: string }[];
  news: { title: string; url: string; source: string; date?: string; snippet?: string }[];
  socialProfileMetrics: {
    instagram?: {
      url?: string; username?: string; followerCount?: number; followingCount?: number;
      postCount?: number; bio?: string; isVerified?: boolean;
      lastPostRecency?: string; engagementIndicator?: 'high' | 'moderate' | 'low' | 'unknown';
    } | null;
    facebook?: {
      url?: string; pageName?: string; followerCount?: number; likeCount?: number;
      rating?: number; reviewCount?: number; bio?: string;
      lastPostRecency?: string; engagementIndicator?: 'high' | 'moderate' | 'low' | 'unknown';
    } | null;
    twitter?: { url?: string; username?: string; followerCount?: number; followingCount?: number; postCount?: number; bio?: string; isVerified?: boolean; } | null;
    tiktok?: { url?: string; username?: string; followerCount?: number; followingCount?: number; videoCount?: number; likeCount?: number; bio?: string; lastPostRecency?: string; engagementIndicator?: string; } | null;
    yelp?: { url?: string; rating?: number; reviewCount?: number; priceRange?: string; categories?: string[]; claimedByOwner?: boolean; } | null;
    summary?: {
      totalFollowers: number;
      strongestPlatform: string;
      weakestPlatform: string;
      overallPresenceScore: number;      // 0-100
      postingFrequency: 'daily' | 'weekly' | 'sporadic' | 'inactive' | 'unknown';
      recommendation: string;
    };
  } | null;
  validationReport: {
    totalUrlsChecked: number;
    valid: number;
    invalid: number;
    corrected: number;
  } | null;

  // --- Backwards-compat mirror (same fields nested under identity) ---
  identity: {
    phone: string | null;
    email: string | null;
    hours: string | null;
    googleMapsUrl: string | null;
    socialLinks: { /* same shape as top-level socialLinks */ };
    logoUrl: string | null;
    favicon: string | null;
    primaryColor: string | null;
    secondaryColor: string | null;
    persona: string | null;
    menuUrl: string | null;
    competitors: { name: string; url: string; reason?: string }[];
  };

  // Per-agent latest outputs (written by writeAgentResult)
  // Keys: "margin_surgeon" | "seo_auditor" | "traffic_forecaster" | "competitive_analyzer"
  //       | "social_profiler" | "discovery_reviewer" | "news_discovery" | "marketing_swarm"
  latestOutputs: {
    [agentName: string]: {
      score: number | null;              // 0-100
      summary: string | null;
      reportUrl: string | null;          // GCS public URL to HTML report
      agentVersion: string;              // e.g. "1.0.0"
      runAt: Timestamp;
      // Agent-specific promoted KPIs (flat, no nesting):
      // margin_surgeon:        total_leakage, menu_item_count
      // seo_auditor:           (scores stored as score field)
      // traffic_forecaster:    peak_slot_score
      // competitive_analyzer:  competitor_count, avg_threat_level
    };
  };

  // CRM state (written by writeInteraction)
  crm?: {
    status: 'outreached' | 'responded' | 'archived';
    outreachCount: number;
    lastOutreachAt: Timestamp;
    lastReportShared: string;            // report URL
    respondedAt?: Timestamp;
    archivedAt?: Timestamp;
    archiveReason?: 'no_response' | 'not_interested' | 'converted';
  };
}
```

**Rules:**
- No arrays that grow over time (no `reports[]`, no `analyses[]`)
- No binary blobs — images go to GCS, store URL only
- `zipCode` is always top-level, never parsed from address at query time
- Use `update()` with dotted paths for nested fields; `set({merge:true})` only for top-level/new docs

## `hub_searches/{autoId}`

Lead capture — logs every search query and email captures.

```typescript
{
  query: string;                         // "Bosphorus Nutley NJ"
  timestamp: Timestamp;
  status: 'pending_email' | 'captured';
  email?: string;                        // set when user submits email
  email_captured_at?: Timestamp;
}
```

## `users/{uid}`

User accounts created on first Google sign-in. `uid` is the Firebase Auth UID.

```typescript
{
  email: string;                         // "user@example.com"
  displayName: string | null;            // Google display name
  photoURL: string | null;               // Google profile picture URL
  createdAt: Timestamp;                  // first login
  lastLoginAt: Timestamp;               // updated every login
  businesses: string[];                  // slugs of businesses they've searched
}
```

## Admin-specific collections

Written by `admin/` only. Web app does NOT read these.

- `workflows` — WorkflowDocument records
- `businesses/{slug}.latestOutputs` — Capability results (written by admin after calling web API)
- `zipcode_research` — ZipCodeReport data
- `area_research` — AreaResearchDocument data
- `sector_research` — SectorResearchDocument data
- `combined_contexts` — CombinedContext data
- `test_fixtures` — TestFixture data for QA

### Persistence mapping (admin → Firestore)

| Capability | Firestore Key | Source Endpoint |
|---|---|---|
| SEO | `latestOutputs.seo_auditor` | `POST /api/capabilities/seo` |
| Traffic | `latestOutputs.traffic_forecaster` | `POST /api/capabilities/traffic` |
| Competitive | `latestOutputs.competitive_analyzer` | `POST /api/capabilities/competitive` |
| Margin Surgeon | `latestOutputs.margin_surgeon` | `POST /api/v1/analyze` |
| Insights | `insights` (top-level) | Generated locally by admin |
