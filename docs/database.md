# Hephae Forge: Database & State Design

Hephae Forge uses a hybrid storage model to balance real-time workflow orchestration with long-term analytical intelligence.

---

## 1. Google Cloud Firestore (Real-Time State)

Firestore is the source of truth for **active workflows** and **business identities**.

### Core Collections:
- **`businesses`**: The primary entity.
    - `identity`: Name, address, website, contact info.
    - `discoveryStatus`: `scanned` | `qualified` | `analyzed` | `approved`.
    - `latestOutputs`: Stores the JSON result of each capability agent (SEO, Margin, etc.).
- **`workflows`**: Tracks the state of a zipcode/area run.
    - `phase`: `DISCOVERY` | `QUALIFICATION` | `ANALYSIS` | `EVALUATION`.
    - `progress`: Aggregated stats (Total biz, Qualified biz, Completed caps).
- **`tasks`**: Manages Cloud Task execution and ADK session IDs.
- **`zipcode_reports`**: Cached macro-research data for specific zipcodes.

---

## 2. Google BigQuery (Longitudinal Intelligence)

BigQuery is used for **benchmarking** and **feedback loops**. It stores data that doesn't need millisecond latency but requires complex relational analysis.

### Core Tables:
- **`capability_outcomes`**: Records every run of an agent.
    - Used to calculate "Signal Effectiveness" (e.g., "Does having a Shopify site correlate with high SEO scores?").
- **`market_benchmarks`**: Aggregated metrics per industry/county.
    - Average star rating, common platforms, median menu prices.
- **`user_feedback`**: Tracks which insights were useful to real human users.

---

## 3. Cloud Storage (Asset Persistence)

GCS is used for "Heavy" or "Public" assets generated by agents.
- **`menus/`**: Screenshots and PDFs of business menus.
- **`reports/`**: Publicly hostable HTML/PDF versions of the intelligence reports sent to owners.
- **`social_cards/`**: Universal social sharing images for outreach.

---

## 4. State Management Principle

**No Overwrites**: We prefer "Dotted Path Updates" (`identity.email`) over replacing entire documents. This ensures that a Discovery agent doesn't accidentally wipe out data previously found by a Qualification agent.
